I’ve been hard at work on the "Login or Sign Up" screen for my **"Cheat Sheet Generator"** project. As a Computer Science student at CSU Chico, I usually focus on the backend code, but I’ve been using Figma to create a **wireframe** -- a skeletal blueprint used to plan the layout and functionality of an app before adding colors or images -- to map out the UI first. Today, I hit a major roadblock while trying to do something that seemed simple: making a button wider. I wanted to update my standard "Sign In" button to a more descriptive call-to-action: "Create a new account and join the community". My goal was for the button's background to grow automatically as I typed, keeping everything neat and centered.

This is where my **Mental Model** -- my internal understanding of how a tool should work based on my past experiences -- clashed with the software. In the physical world, if you put a long sticker on a box, the box doesn't magically grow; you just have to get a bigger box. I expected Figma to "know" I wanted the button to expand. Instead, I encountered a **Constraint**, which is a built-in rule that limits how an element can behave. As you can see in my first recording, the button stayed a fixed width, which caused my new, longer text to clip off the edges and look broken.

I felt stuck because the **Signifiers** -- the visual cues like icons or buttons that tell you how to use a tool -- weren't obvious. I had to go hunting through the side menus to find the **Auto Layout** settings. Initially, I accidentally selected the entire login form instead of just the button, causing a "layout explosion" where the entire screen turned into a giant white box. I had to use **Error Recovery**, which is the ability to fix a mistake in a system, by hitting `Ctrl + Z`. Eventually, I found the alignment grid in the properties panel. As shown in the second clip, once I changed the alignment from "Justify" to "Center," I finally got the **Feedback** I was looking for. **Feedback** is the system's way of telling the user that their action was successful; in this case, the text finally snapped to the middle where it belonged.

While Figma’s Auto Layout is powerful because it offers precise control, its weakness is that it often forces a **Conceptual Model** -- the actual logic the designers built into the tool -- that feels "unnatural" to beginners. It requires too many manual steps for a behavior that should be the default. I suggest that Figma should implement a "Smart Default". If a user centers a text layer inside a rectangle and then turns on Auto Layout, the software should automatically set the alignment to "Center" instead of forcing the user to dig through a sidebar to fix it. This would better support the usability principle of **"Match between system and the real world"** by making the software behave more like we expect physical objects to.
